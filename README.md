# ①課題名
楽しかったことを記録するツール

## ②課題内容（どんな作品か）
- ボタンをクリックすると楽しかったことを記録できるようにするツールです。
- 削除ボタンと更新ボタンを追加しました。

## ③アプリのデプロイURL
http://blue-purge.sakura.ne.jp/index.php

## ④アプリのログイン用IDまたはPassword（ある場合）
- ID: NA
- PW: NA

## ⑤工夫した点・こだわった点
- セキュリティの観点から、いくつか設計を見直しました。
- ⓵データベース名・ユーザー名・パスワードなどの接続情報を 公開ディレクトリであるwwwのと同階層に作ったconfigフォルダへ分離し、直接URLから参照できない構成にしました。
- ⓶エラーログの詳細をサーバーサイドに残すようにし、セキュリティを高めました。catchのあとのexitの処理をサーバサイドに残るようにしています。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- データベース接続やエラー処理について、「本番ではどうあるべきか」を意識するのが難しかったです。
- 特に、try-catch や error_logなどは、これまであまり深く考えずに使っていたため、ユーザーに何を見せて、サーバ側には何を残すべきかを整理するのに時間がかかりました。
